## 8. 품질 검증 체크리스트 (실행 전)
> **내용의 질에 집중.** 양식 빈칸 충실도가 아니라 "Plan이 좋은 Source 결과를 낳을 만한가"를 검증.
> 빈칸 자체는 양식이 보여주니 별도 체크 불필요.

- [ ] 키워드가 번역체가 아닌 **원어 그대로**인가
- [ ] T1(1차) 자료원 2개 이상이 **서로 보완적**인가 (같은 출처의 중복 아님)
- [ ] 모든 Tier가 채워져 있고 한쪽 Tier에 쏠림 없는가
- [ ] 자료원이 모두 **구체적**인가 (`Google`, `유튜브` 같은 추상 자료원 없음. `site:` 검색식은 허용)
- [ ] 최신성이 주제 성격(법규/시장/기술/기초)에 맞는가
- [ ] 핵심 질문이 **자료로 검증 가능**한가 (미래 예측·가치 판단 X)
- [ ] 제외 키워드가 **실제** 노이즈를 차단하는가
- [ ] NotebookLM 실행 지시(섹션 7)가 추가 질문 없이 따라할 수 있는가 (Source 팀 입장 시뮬레이션)
- [ ] "알 필요 없는 것"(섹션 2-3)이 1개 이상 명시되어 있는가
