# 자가 진화 시스템 (Self-Evolving Research System)

## 1. 핵심 통찰

```
다른 리서치 도구:
- 기능 고정
- 모두 같은 출력
- 시간이 지나도 그대로

NEXUS:
- 사용자별 API 카탈로그 확장
- Gap Report → API 발급 → 시스템 진화 루프
- 쓸수록 그 사람 전용 시스템이 됨
```

이건 lock-in 효과를 만들면서 동시에 사용자 가치를 높이는 구조.
드물게 양쪽이 같은 방향을 가리킴.

---

## 2. 사용자 여정과 시스템 진화

| 단계 | 사용자 행동 | 시스템 상태 | 리서치 품질 |
|---|---|---|---|
| 1회차 | 첫 리서치 | Core API만 | 기본 |
| 1회차 후 | Gap Report 확인 | "특허청 API 권장" 안내 | - |
| 2회차 직전 | 사용자가 API 발급/등록 | Core + 특허청 | - |
| 2회차 | 리서치 실행 | 자동으로 새 API 활용 | 향상 |
| ... | 반복 | 도메인 API 누적 | 압도적 |
| 5회차 | 리서치 실행 | 그 사용자만의 풍부한 카탈로그 | 다른 도구 대비 압도적 |

같은 플러그인이지만 **6개월 쓴 사용자와 신규 사용자의 결과 품질이 다름**.
이게 진짜 차별화.

---

## 3. 새로 필요한 설계 요소

### A. 사용자 API 등록 메커니즘

| 요소 | 설계 |
|---|---|
| API 키 저장소 | 로컬 안전 저장 (.env / keychain) |
| 등록 인터페이스 | CLI 명령 또는 설정 파일 |
| 자동 인식 | Plan 시스템이 등록된 API를 자동으로 sources_map에 포함 |
| 권한 관리 | API별 사용 한도, 호출 빈도 관리 |

### B. API 진화 트래킹

| 요소 | 설계 |
|---|---|
| 누적 카탈로그 | 사용자가 등록한 API 목록 |
| 사용 통계 | 각 API 호출 빈도, 자료 기여도 |
| 품질 향상 지표 | API 추가 전후 리서치 품질 비교 |
| 다음 추천 | "이 분야면 다음엔 X API 추천" |

### C. Gap Report의 위상 변화

기존: 단순 안내 보고서
변경: **시스템 진화 가이드**

```markdown
## 이번 리서치의 갭

❌ 해외 특허 동향 누락
   → Google Patents API 추가 시 즉시 해결
   → 발급: 무료/즉시
   → 효과: 글로벌 특허 리서치 가능

❌ 임상 데이터 부족
   → ClinicalTrials.gov 추가 시 해결
   
📊 당신의 시스템 현황
   - 등록 API: 4개
   - 활용도: 78%
   - 다음 추가 시 가장 큰 향상: Google Patents
```

Gap Report = 사용자에게 "당신 시스템을 어떻게 강화할지" 보여주는 대시보드.

## 4. 도메인 프로파일과의 결합

도메인 프로파일은 **시작점**.
자가 진화는 **그 위에 누적**.

```
변리사 사용자:
  시작: 변리사 프로파일 (특허청 API 기본 포함)
  1개월: + Google Patents (Gap Report 안내로 추가)
  3개월: + WIPO 검색 API
  6개월: + 유료 특허 분석 도구 API
  
  → 시중 어떤 변리사 리서치 도구보다 강력
```

도메인 프로파일은 출발점을 결정하고,
자가 진화는 깊이를 결정함.

## 5. 마케팅 메시지 진화

| 단계 | 메시지 |
|---|---|
| 기본 도구 | "AI로 리서치 자동화" |
| 도메인 특화 | "당신 분야에 맞춘 리서치" |
| **자가 진화 추가** | **"쓸수록 깊어지는 리서치"** |

가장 강력한 메시지:
```
"하루 쓴 도구와 6개월 쓴 도구는 다릅니다.
 당신만의 리서치 시스템으로 진화합니다."
```

---

## 6. 비즈니스 모델 함의

자가 진화 시스템은 자연스러운 가격 정당화 근거가 됨:

| 시점 | 사용자 인식 |
|---|---|
| 신규 사용자 | "기본 도구다" → 낮은 가격 OK |
| 1개월 사용자 | "내 데이터, 내 카탈로그 쌓임" → 이탈 비용 발생 |
| 6개월 사용자 | "다른 도구로 못 옮김. 내 시스템임" → 가격 둔감 |

구독형 모델의 lock-in 동력이 자연스럽게 형성됨.
사용자가 "갇혔다"고 느끼지 않고 "내 자산이다"라고 느낌. 이게 핵심.

## 7. 결정이 필요한 항목

| 번호 | 항목 |
|---|---|
| E-1 | API 키 저장 방식 (로컬 .env? OS keychain? 별도 vault?) |
| E-2 | 사용자 API 등록 인터페이스 (CLI 명령? GUI? 설정 파일?) |
| E-3 | 진화 통계를 어디까지 보여줄지 (간단 알림? 대시보드?) |
| E-4 | Gap Analyzer가 추천하는 API DB는 어디서 관리? (플러그인 내장? 원격 업데이트?) |
| E-5 | 도메인 프로파일의 "기본 API 세트"와 사용자 추가 API의 우선순위 규칙 |

## 8. 한 줄 정리

```
NEXUS는 리서치 도구가 아니라
사용자별 리서치 시스템 생성기다.

플러그인을 깔면 도구를 얻는 게 아니라
당신만의 리서치 OS를 시작하는 것이다.
```
