# 검색 플랜 품질 체크리스트

완성된 검색 플랜 파일을 Source 팀에 넘기기 전 자가 검증용.
**내용의 질에 집중** — 양식 빈칸 충실도는 양식 자체가 이미 보여주므로 별도 체크하지 않는다.
**모두 ✓ 후 전달.** 미통과 항목 3개 이상이면 Plan을 처음부터 재작성.

---

## A. 내용 품질 (Plan 자체가 좋은가)

### 제1원칙: 사실관계 마커 (`guides/authoring_guide.md` 제1원칙 섹션 참조)
- [ ] 직접 1차 출처를 확인하지 못한 사실값(버전·통계·날짜·분류·자료원 성격 등)에 모두 `[?]` 부착
- [ ] 마커가 `[?]` **단일 형태**로 통일됨 (`[모름]`, `[검증 필요]`, `TBD` 혼용 없음)
- [ ] 30초 룰 적용: 30초 안에 확인 가능한 값은 직접 확인 후 마커 없이 적음 / 그 외는 `[?]`
- [ ] LLM 작성자인 경우 training data 회상은 직접 확인으로 간주하지 않았는가 (WebSearch/WebFetch/공식 URL 직접 호출 후만 마커 제거)

### 주제·목적
- [ ] 주제가 한 줄로 끝남 (시점·지역·대상 명시)
- [ ] 목적이 "결정/산출물" 단위로 적힘 ("이해하기 위해"는 X)
- [ ] "알 필요 없는 것"(섹션 2-3)이 1개 이상 명시됨

### 핵심 질문
- [ ] 5~10개 범위
- [ ] 모두 자료로 답 가능 (미래 예측·가치 판단형 X)

### 키워드
- [ ] 검색이 실제 이뤄지는 언어의 **원어 그대로** (번역체 X)
- [ ] 영어 키워드 최소 3개 (한국어 주제여도 필수)
- [ ] Primary / Secondary / Long-tail 구분이 의미 있게 채워짐
- [ ] 제외 키워드가 **실제** 노이즈를 차단함

### 소스 매트릭스
- [ ] T1(1차) 자료원 2개 이상이 **서로 보완적**인가 (같은 출처 중복 아님)
- [ ] T2 / T3 각각 자료원 있음 (한 Tier 쏠림 없음)
- [ ] 자료원이 모두 구체적 (`Google`·`유튜브` 같은 추상 자료원 없음, `site:` 검색식은 허용)
- [ ] 묶음 자료원은 각 자료원이 식별 가능하게 표기됨
- [ ] **`API 필요 여부` 컬럼이 모든 행에 기록됨** (`불필요` / `[API 필요: 도구·키]` / `[? Source 팀 결정]` 중 하나)
- [ ] 자료원 성격에 대한 단정("X는 RN 글 거의 없음" 등)에 직접 확인이 없으면 `[?]` 부착되어 있는가

### 검색 파라미터
- [ ] 언어 가중치가 주제 성격에 맞음 (예: 한국 법규 = KO 우세)
- [ ] 최신성이 주제 성격에 맞음 (법규=6개월, 기초개념=무관)
- [ ] KO 자료가 T3(개인 블로그)에만 있지 않음 (KO T2 최소 1개)

### NotebookLM 실행 지시
- [ ] Notebook 이름 규칙 따름: `[YYYY-MM-DD][도메인] [주제]`
- [ ] 소스 추가 순서가 T1 → T2 → T3
- [ ] 소스 메타 라벨링 형식 명시
- [ ] 추가 질문 없이 따라할 수 있는가 (Source 팀 입장 시뮬레이션)

### Gap Report
- [ ] 빈 골격(섹션 9)이 만들어져 있음 — 실행 후 Source 팀이 채울 수 있도록
- [ ] 9-5 (Plan 사실관계 검증 결과) 골격이 포함되어 있는가 — 작성자 단정 vs NotebookLM 자료 일치 점검용

---

## B. 메타 (양식 자체의 운영 품질)

- [ ] **Plan 작성 30분 이내**에 완료 (Source 팀의 실제 검색·적재 시간은 별도)
  - 넘으면: 양식 과함 또는 주제 분해 부족 → 섹션 2 재작성
- [ ] 같은 양식으로 다른 주제도 채울 수 있는가
  - 못 채우면: 이번 주제용 변형이 섞임 → 표준 양식으로 복귀

---

## C. 빠지기 쉬운 함정

- [ ] **뉴스 쏠림**: T2에만 몰리지 않았는가 → T1·T3 균형 확인
- [ ] **언어 쏠림**: 한국어 자료만 모으는 함정에 빠지지 않았는가 → 영어 자료 비중 확인
- [ ] **최신성 과단**: 3개월만 잡아 기초 개념을 놓치지 않는가
- [ ] **최신성 과장**: 5년+ 잡아 낡은 시장·법규 자료가 섞이지 않는가
- [ ] **자료원 추상화**: "구글", "유튜브", "공식 사이트"처럼 모호하게 적지 않았는가
- [ ] **핵심 질문 의견화**: "좋은가/나쁜가" 같은 가치 판단형이 섞이지 않았는가
- [ ] **제외 영역 누락**: "알 필요 없는 것"이 비어 있지 않은가
- [ ] **추정 단정**: 학습 데이터 기억으로 버전·수치·자료원 성격을 단정하지 않았는가 (`[?]` 누락 없음)
- [ ] **LLM 자기 면제**: LLM 작성자가 "그냥 알아서" 마커 없이 적은 사실값 없는가

---

## D. 미통과 시 처리

| 미통과 항목 수 | 처리 |
|---|---|
| 1~2개 | 해당 섹션만 보강 후 재검증 |
| 3~5개 | Plan 절반 이상 재작성. 주제 분해(섹션 2)부터 다시 |
| 6개 이상 | 주제 자체를 재정의 (섹션 1로 복귀) |

---

## E. 검증 통과 후

- 파일을 `output/[YYYY-MM-DD][도메인] [주제].md` 형식으로 저장 (이미 그 위치에 작성했다면 추가 작업 없음)
- Source 팀에 전달
- 실행 후 섹션 9 Gap Report를 Source 팀이 채워 반환받음 → 다음 플랜의 도메인 프로파일 입력
